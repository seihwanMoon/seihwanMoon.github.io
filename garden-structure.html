Garden Structure

이 문서는 github.vault 가든의 공개 구조를 설명한다. 이 가든은 개인 작업 로그 전체를 그대로 노출하는 공간이 아니라, 시간이 지나도 읽을 가치가 있는 기술 노트와 실험 기록을 주제별로 엮어 보여주는 공개용 디지털 가든이다.

구조는 고정된 설계도가 아니라 점진적으로 자라는 지도에 가깝다. 다만 탐색이 흔들리지 않도록 몇 가지 상위 허브와 배치 원칙은 비교적 안정적으로 유지한다.

이 가든의 기본 방향

  • 최근 갱신과 주제 탐색을 동시에 제공하는 디지털 가든
  • 공개 가능한 노트만 포함하는 공개 전용 vault
  • 도구 이름보다 노트가 답하는 질문을 우선하는 배치 원칙
  • 작은 수정 기록보다 읽을 가치가 있는 변화를 남기는 업데이트 운영

루트 페이지

루트에는 방문자가 가든을 이해하고 탐색을 시작하는 데 필요한 핵심 페이지만 둔다.

  • Home
    최근 갱신과 대표 허브를 함께 보여주는 첫 화면
  • Topics
    전체 주제 허브를 모아 보여주는 탐색 지도
  • Updates
    공개할 가치가 있는 변화만 모은 큐레이션 타임라인
  • Now
    현재 집중하고 있는 관심사와 읽는 흐름을 짧게 보여주는 페이지
  • About
    이 가든이 다루는 문제의식과 범위를 설명하는 소개 페이지
  • Garden-Structure
    이 가든이 어떤 허브와 원칙으로 구성되는지 설명하는 구조 안내서

상위 허브

이 가든의 상위 허브는 도구와 주제를 함께 반영하되, 너무 세분화하지 않는 수준으로 시작한다.

coding-systems

개발 자체를 더 잘하기 위한 노트를 둔다. 코딩 습관, 개발 워크플로, 코드 품질, 리팩토링, 도구 활용법, 생산성 개선 같은 주제가 여기에 들어간다.

예시:

  • 코딩 워크플로
  • 개발 생산성
  • 코드 품질
  • 리팩토링 메모

ai-agents

에이전트를 어떻게 설계하고 운용할지에 대한 노트를 둔다. tool use, memory, orchestration, multi-agent, 역할 분리, 상태 관리 같은 주제를 다룬다.

예시:

  • agent design
  • tool use
  • multi-agent
  • memory and context

llm-platforms

특정 도구나 플랫폼 자체를 다루는 노트를 둔다. codex, claude, openclaw 같은 하위 영역과 비교 노트를 여기에 모은다.

예시:

  • codex
  • claude
  • openclaw
  • comparisons

automation

반복 업무를 줄이고 흐름을 자동화하는 노트를 둔다. 업무자동화, 개인 자동화, 스크립트, 파이프라인, 에이전트 기반 반복 작업 등이 여기에 속한다.

예시:

  • work automation
  • personal automation
  • scripts and pipelines

ml-dl

모델, 학습, 추론, 평가에 관한 노트를 둔다. 이론과 실무를 함께 다루되, 핵심 질문이 모델링 자체에 있을 때 이 허브에 둔다.

예시:

  • fundamentals
  • training
  • inference
  • evaluation

industrial-automation

스마트공장, 제조자동화, 산업 시스템 적용에 관한 노트를 둔다. 현장 적용, 검사, 설비, 생산 흐름, 산업 AI 같은 주제가 여기에 들어간다.

예시:

  • smart factory
  • manufacturing automation
  • industrial AI
  • vision and inspection

notes-and-methods

이 가든 전체를 관통하는 생각법과 작업법을 둔다. 내부 운영 매뉴얼이 아니라, 공개해도 의미 있는 메타 노트만 둔다.

예시:

  • workflows
  • experiments
  • playbooks
  • note-taking methods

각 상위 허브는 최소한 index.md 를 가진다. 이 페이지는 해당 허브의 입구이며, 핵심 노트와 읽는 흐름을 보여주는 허브 노트가 된다.

노트 타입

이 가든에서는 노트 타입을 너무 많이 만들지 않는다. 아래 다섯 가지면 충분하다.

  • hub note
    어떤 영역의 입구가 되는 허브 페이지
  • concept note
    오래 가는 개념 설명 노트
  • tool/platform note
    특정 도구나 플랫폼을 다루는 노트
  • guide note
    실제 사용법, 워크플로, 적용 방법을 정리한 노트
  • update note
    공개할 가치가 있는 변화나 새 실험을 정리한 노트

노트를 어디에 둘지 판단하는 방법

노트의 제목보다 이 노트가 답하는 핵심 질문을 먼저 본다.

  • 이 도구가 무엇이고 어떻게 다른가: llm-platforms
  • 에이전트를 어떻게 설계할까: ai-agents
  • 개발 생산성을 어떻게 높일까: coding-systems
  • 반복 업무를 어떻게 자동화할까: automation
  • 모델을 어떻게 학습, 평가, 추론할까: ml-dl
  • 제조나 현장에 어떻게 적용할까: industrial-automation
  • 이 가든을 어떤 방식으로 읽고 만들까: notes-and-methods

예를 들어 Codex vs Claude 비교는 llm-platforms/comparisons 에 두는 것이 자연스럽다. 반면 Codex를 활용한 코딩 워크플로는 핵심 질문이 개발 방식에 있으므로 coding-systems 쪽에 둘 수 있다.

중복을 줄이는 원칙

같은 내용을 여러 허브에 복사하지 않는다. 하나의 정본 노트를 두고, 다른 허브에서는 링크로 연결한다.

예시:

  • 도구 비교 노트는 llm-platforms
  • 에이전트 설계 원칙 노트는 ai-agents
  • 특정 도구를 활용한 개발 워크플로 노트는 coding-systems
  • 제조 현장 적용 사례 노트는 industrial-automation

업데이트 운영 원칙

이 가든은 최근 갱신을 보여주지만, 모든 수정을 타임라인으로 남기지는 않는다.

업데이트로 남길 만한 변화:

  • 새 허브나 중요한 비교 노트 추가
  • 큰 구조 개편
  • 읽는 흐름이 달라질 정도의 시리즈 정리
  • 새로운 실험 방향이나 중요한 관찰

업데이트로 남기지 않는 변화:

  • 오탈자 수정
  • 작은 링크 정리
  • 문장 다듬기
  • 방문자 입장에서 거의 의미가 없는 사소한 개편

즉 Updates 는 운영 로그가 아니라, 가든이 어떻게 자라고 있는지 보여주는 공개 큐레이션 페이지다.

홈의 역할

Home 은 긴 소개문보다 탐색을 시작시키는 허브에 가깝게 구성한다.

추천 흐름:

  1. 짧은 소개
  2. 최근 갱신
  3. 대표 허브 목록
  4. 지금 읽기 좋은 노트
  5. Topics, Now, About 으로 이어지는 링크

이렇게 하면 처음 방문한 사람도 최신 변화와 전체 구조를 동시에 이해할 수 있다.

구조는 바뀔 수 있다

이 구조는 완성된 도면이 아니라 현재 시점의 운영 계약이다. 관심사가 바뀌거나 노트가 쌓이면 허브와 서브토픽은 조정될 수 있다.

다만 아래 원칙은 유지하는 편이 좋다.

  • 상위 허브는 자주 흔들지 않는다
  • 서브토픽은 필요에 따라 유연하게 바꾼다
  • 개별 노트는 더 적절한 위치로 이동할 수 있다
  • 구조가 바뀌면 Topics 와 Garden-Structure 도 함께 갱신한다

좋은 구조는 처음부터 완벽한 구조가 아니라, 탐색을 방해하지 않으면서 자연스럽게 확장되는 구조다.

초기 시작 방식

처음부터 모든 관심사를 다 채우기보다, 아래 순서로 천천히 시작하는 편이 좋다.

  1. 루트 핵심 페이지를 먼저 만든다
  2. 각 상위 허브의 index.md 를 만든다
  3. 대표 노트가 생기는 허브부터 채운다
  4. llm-platforms, ai-agents, coding-systems, automation 을 우선적으로 확장한다
  5. ml-dl, industrial-automation 은 허브부터 만들고 점진적으로 넓힌다

이 가든은 완성본보다 성장 과정을 보여주는 구조에 가깝다. 중요한 것은 모든 것을 한 번에 채우는 것이 아니라, 읽을 가치가 있는 노트와 연결을 꾸준히 쌓아가는 것이다.
