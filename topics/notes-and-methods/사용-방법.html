사용 방법

이 노트는 github.vault를 읽고 확장하는 기본 흐름을 정리한다. 운영 매뉴얼 전체를 공개하는 대신, 이 가든을 어떤 방식으로 다루는지가 보이도록 최소한의 원칙만 남긴다.

이 볼트의 역할

이 볼트는 단순 메모장이 아니라, 다음 흐름으로 굴러가는 LLM-maintained wiki다.

  • 사람은 원문을 raw/에 넣는다
  • 에이전트는 루트 허브와 topics/에 durable note를 만든다
  • 가치 있는 질의 결과는 outputs/, 루트 허브, 또는 topics/에 저장한다
  • 운영 규칙은 AGENTS에서 관리한다

폴더별 사용법

  • raw/
    • 원문 보관용이다
    • 웹문서, PDF, 메모, 클립 원문을 넣는다
    • 원문은 가능하면 수정하지 않는다
  • 루트 허브와 topics/
    • 정리된 지식 노트를 둔다
    • 루트는 읽기 흐름을, topics/는 주제별 지식을 담당한다
    • source note, 개념 노트, 주제 노트, 종합 노트가 여기에 쌓인다
  • outputs/
    • 일단 저장할 가치가 있는 결과물을 둔다
    • 발표 초안, 비교표, 일회성 분석 결과가 여기에 들어간다
  • scratch/
    • 임시 메모와 실험 노트를 둔다
    • 아직 정식 지식이 아닌 내용은 여기서 다룬다
  • .codex-obsidian/
    • Codex 설정, 로컬 명령, 스킬, 세션 파일을 둔다

기본 사용 순서

1. 원문 넣기

새로 읽고 싶은 문서나 참고 자료를 raw/에 넣는다.

예:

  • 웹문서 복사본
  • gist raw markdown
  • PDF
  • 개인 메모 초안

2. ingest 요청하기

나에게 원문 링크나 파일을 주고 ingest를 요청한다.

예:

  • "이 링크를 첫 source로 ingest해줘"
  • "raw 폴더의 이 PDF를 요약해서 위키 노트로 만들어줘"

그러면 나는 보통 다음을 수행한다.

  • 원문을 읽는다
  • 관련 노트가 있는지 본다
  • topics/ 또는 루트 허브에 source note 또는 concept note를 만든다
  • Home.md, Topics.md, index.md 중 필요한 페이지를 갱신한다
  • log.md에 ingest 기록을 남긴다

3. 질문하기

위키가 어느 정도 쌓이면 질문을 던진다.

예:

  • "지금까지 들어온 source 기준으로 핵심 주장 비교해줘"
  • "이 개념을 한국어로 설명해줘"
  • "이 주제에 대한 topic page 만들어줘"

질문 결과가 오래 쓸 가치가 있으면:

  • topics/에 concept/topic/synthesis note로 저장하거나
  • outputs/에 결과물로 저장한다

4. 정리 점검하기

노트가 몇 개 쌓이면 lint를 요청한다.

예:

  • "이 볼트 lint 해줘"
  • "고아 노트나 겹치는 노트 있는지 점검해줘"

그러면 다음을 본다.

  • 링크가 약한 노트
  • 모순되는 주장
  • 낡은 내용
  • 승격이 필요한 outputs/ 또는 scratch/ 노트

추천 작업 패턴

처음에는 이 3단계를 반복하는 것이 가장 좋다.

  1. source 넣기
  2. ingest 하기
  3. 질문해서 종합 노트 만들기

이 흐름이 몇 번 반복되면 그때:

  • topic hub를 보강하고
  • concept 페이지를 분리하고
  • 필요하면 topics/ 하위 폴더를 더 만든다

Codex에게 요청하는 좋은 방식

좋은 요청:

  • "이 링크를 ingest해줘"
  • "이 두 source를 비교해서 정리해줘"
  • "이 결과를 outputs 말고 topics에 남겨줘"
  • "현재 주제 노트 기준으로 topic page 만들어줘"
  • "lint 돌려서 구조 점검해줘"

덜 좋은 요청:

  • "대충 정리해줘"
  • "알아서 다 해줘"

좋은 요청일수록 결과물이 더 durable하고 재사용 가능해진다.

자주 보는 파일

  • Home: 메인 진입점
  • Topics: 전체 주제 지도
  • index: 내부 카탈로그
  • log: 작업 이력
  • AGENTS: 운영 규칙

현재 권장 다음 단계

  • source를 2~3개 더 넣어서 비교 가능한 상태를 만든다
  • llm-platforms, ai-agents, automation 허브부터 실제 노트를 채운다
  • 그 다음 lint를 돌려 구조를 다듬는다
