LLM-Maintained Wiki Pattern

이 노트는 Source - Karpathy - LLM Wiki에서 제안한 핵심 패턴을 정리한다.

정의

LLM-maintained wiki는 raw source와 이후 질문들 사이에 놓이는 지속적인 지식 계층이다. raw 문서는 그대로 유지되고, LLM은 현재의 이해 상태를 나타내는 상호 연결된 마크다운 노트를 계속 쓰고 갱신한다.

단순 RAG와의 차이

표준적인 RAG 워크플로에서는 질문 시점에 관련 청크를 찾아 답을 재구성한다. 이런 방식도 동작하지만, 지식이 유지보수되는 산출물로 누적되지는 않는다.

LLM-maintained wiki 워크플로에서는 다음이 가능하다.

  • 종합 해석이 지속적으로 남는다
  • 교차 링크를 시간이 지나며 개선할 수 있다
  • 모순을 한 번 표시한 뒤 계속 추적할 수 있다
  • 좋은 답변을 채팅 기록에 묻지 않고 위키로 승격할 수 있다

핵심 변화는 반복 검색에서 지속적 컴파일과 유지보수로의 전환이다.

핵심 역할

  • Human: 원문을 선별하고, 강조점을 정하고, 더 좋은 질문을 던지고, 무엇이 중요한지 판단한다
  • LLM: 요약하고, 연결하고, 수정하고, 조정하고, 위키의 일관성을 유지한다

최소 시스템 형태

  • raw/: immutable source layer
  • wiki/: maintained markdown notes
  • AGENTS.md: operating schema for the agent
  • index.md: catalog for discovery
  • log.md: chronological record of change

운영 사이클

Ingest

새 원문을 읽고, 무엇이 달라졌는지 확인하고, durable note를 갱신하고, ingest를 기록한다.

Query

먼저 유지보수된 위키에서 답한다. 결과가 durable하면 노트나 산출물로 저장한다.

Lint

구조적 약점, 낡은 주장, 누락된 노트를 점검한 뒤 위키를 보수한다.

설계상 함의

  • 구조는 최소한으로 시작하고 반복 사용을 통해 진화해야 한다
  • 좋은 index가 있으면 무거운 검색 도구 도입 시점을 늦출 수 있다
  • 출력 형식은 markdown, 표, 캔버스, 차트, 슬라이드처럼 유연하게 유지할 수 있다
  • 유지보수 비용이 충분히 낮아질 때 위키는 지속 가능한 자산이 된다

이 볼트에서의 의미

이 패턴 때문에 이 볼트는 raw/를 읽기 전용으로 두고, AGENTS를 schema 계층으로 사용하며, index와 log를 핵심 운영 파일로 유지한다.
