github.vault LLM Wiki 운영 규칙

이 볼트는 "LLM이 유지보수하는 위키" 패턴을 따른다.

  • raw/는 불변 원문 계층이다.
  • 루트 허브 페이지와 topics/는 유지보수되는 공개 지식 계층이다.
  • AGENTS.md, Home.md, Topics.md, index.md, log.md는 운영 계약이다.
  • outputs/는 질의 결과를 바로 topics/ 또는 허브 노트로 승격하기 전 보관하는 위치다.
  • scratch/는 임시 작업 공간이며 정식 지식으로 간주하지 않는다.

시작 규칙

codex-cli로 이 볼트에서 작업할 때는 항상 루트에서 시작한다.

Set-Location "G:\My Drive\Bobsidian\github.vault"

작업 시작 시에는 다음 순서를 따른다.

  1. Read AGENTS.md.
  2. Read Home.md, Topics.md, index.md.
  3. 질문에 답할 때는 raw/보다 루트 허브 페이지와 topics/를 우선한다.
  4. 허브와 주제 노트에 정보가 없거나, 새 원문이 들어왔거나, 특정 주장을 검증해야 할 때만 raw/를 읽는다.

디렉터리 계약

  • raw/
    • 사람이 수집한 원문 자료다.
    • 읽기 전용으로 취급한다.
    • 명시적 요청이 없으면 원문을 고쳐 쓰거나, 그 자리에서 요약하거나, 과하게 이름을 바꾸지 않는다.
  • 루트 허브 페이지
    • Home.md, Topics.md, Updates.md, Now.md, About.md, Garden-Structure.md, index.md 같은 상위 탐색 페이지다.
    • 방문자의 읽기 흐름과 전체 지도를 제공한다.
    • 구조 개편이나 중요한 연결 변화가 생기면 함께 갱신한다.
  • topics/
    • 유지보수되는 마크다운 노트 계층이다.
    • 주제 허브와 durable note를 둔다.
    • 생성, 수정, 병합, 분리, 교차 링크를 자유롭게 할 수 있다.
    • 기계식 ID보다 사람이 읽기 쉬운 페이지 제목을 선호한다.
  • outputs/
    • 재사용 가능한 답변, 비교표, 슬라이드 초안, 표, 캔버스, 차트를 둔다.
    • 내용이 장기적으로 유효한 지식일 때만 topics/ 또는 루트 허브로 승격한다.
  • scratch/
    • 임시 노트, 부분 분석, 스테이징 문서를 둔다.
    • 작업 중에는 써도 되지만 정식 지식은 아니다.
  • .codex-obsidian/
    • 이 볼트 전용 Codex 설정, 재사용 프롬프트, 로컬 스킬, 세션 파일을 둔다.

기본 작업 흐름

Ingest

새 항목이 raw/에 추가되면 이 흐름을 사용한다.

필수 동작:

  1. raw/에서 새 원문을 읽는다.
  2. 루트 허브 또는 topics/에 관련 페이지가 이미 있는지 확인한다.
  3. 하나 이상의 주제 노트를 만들거나 갱신한다.
  4. 새 지식을 찾을 수 있게 Home.md, Topics.md, index.md 중 필요한 페이지를 갱신한다.
  5. log.md와 필요하면 Updates.md에 항목을 추가한다.

선호하는 노트 형태:

  • topics/ 안의 source summary 페이지
  • 기존 주제 흐름이 바뀌는 경우 topic 페이지 갱신
  • 의미 있는 사실이 추가될 경우 entity 또는 concept 페이지 갱신

ingest 중 확인할 질문:

  • 이 원문이 기존 위키에 없던 무엇을 추가하는가?
  • 현재 노트와 충돌하는 주장이 있는가?
  • 이제 서로 연결되어야 할 페이지는 무엇인가?
  • overview 성격의 페이지에 영향을 줄 만큼 중요한가?

Query

볼트의 내용을 바탕으로 질문에 답할 때 사용한다.

필수 동작:

  1. 먼저 Home.md, Topics.md, index.md를 읽는다.
  2. topics/에서 필요한 최소한의 페이지 집합을 읽는다.
  3. 유지보수된 노트들을 바탕으로 답을 종합한다.
  4. 허브와 주제 노트만으로 부족하거나 검증이 필요할 때만 raw/를 읽는다.
  5. 결과가 지속 가치가 있으면 outputs/, topics/, 또는 적절한 루트 허브에 저장한다.

다음에 해당하면 query 결과를 topics/ 또는 루트 허브로 승격한다.

  • 재사용 가능한 종합 정리
  • 안정적인 비교 자료
  • 오래 두고 볼 설명
  • 기존 주제들 사이의 새로운 연결

다음에 해당하면 query 결과를 outputs/에 둔다.

  • 발표 전용
  • 임시 결과
  • 특정 작업에만 강하게 묶임
  • 아직 정식 위키에 넣기엔 안정적이지 않음

Lint

볼트 상태를 점검하기 위해 주기적으로 사용한다.

다음을 점검한다.

  • 링크가 약하거나 거의 없는 고아 페이지
  • 노트 사이의 상충하는 주장
  • 더 새로운 원문에 의해 낡아진 주장
  • 언급은 되지만 독립 페이지가 없는 중요한 개념/엔티티
  • outputs/에서 topics/ 또는 허브 노트로 승격해야 하는 노트
  • scratch/에서 삭제, 승격, 보관 판단이 필요한 노트

lint 이후에는 다음을 수행한다.

  1. 안전하고 확신이 높은 유지보수 수정은 바로 반영한다.
  2. log.md에 기록한다.
  3. 남는 공백이나 의문점은 명시적으로 적는다.

PowerShell 친화 명령

최근 로그 보기:

Select-String '^## \[' .\log.md

주제 노트 안에서 문구 검색:

Get-ChildItem .\topics -Recurse -File | Select-String 'search term'

세션 파일을 제외한 모든 마크다운 검색:

Get-ChildItem . -Recurse -File -Include *.md |
  Where-Object { $_.FullName -notmatch '\\.codex-obsidian\\sessions\\' } |
  Select-String 'search term'

최근 원문 파일 보기:

Get-ChildItem .\raw -Recurse -File | Sort-Object LastWriteTime -Descending | Select-Object -First 20

작성 규칙

  • 기본 작성 언어는 한국어로 한다.
  • 원문 제목, 고유명사, 정확한 인용구, 명령어, 파일명은 필요할 때 원문 언어를 유지한다.
  • Obsidian flavored markdown을 사용한다.
  • 볼트 내부 링크는 가능하면 [[Home]] 같은 위키링크를 사용한다.
  • 시스템 폴더와 운영 파일 이름은 ASCII로 유지한다.
  • v1에서는 과도한 frontmatter를 피한다. 탐색이나 도구화에 실제로 도움이 될 때만 추가한다.
  • 채팅 기록 같은 제목보다 짧고 오래 가는 페이지 제목을 선호한다.
  • source 성격의 페이지는 제목만 봐도 원문 노트임이 드러나게 만든다.

안전 규칙

  • scratch/를 정식 지식으로 취급하지 않는다.
  • 일반 유지보수 과정에서 raw/ 파일을 덮어쓰지 않는다.
  • 겉보기에 안 쓰는 노트라고 바로 삭제하지 않는다. 먼저 이동, 병합, 표시를 검토한다.
  • 구조적 변경을 할 때는 Home.md, Topics.md, index.md, log.md를 같이 최신 상태로 맞춘다.

초기 가정

  • Git은 필수가 아니다.
  • 검색은 Home.md, Topics.md, index.md, Obsidian 검색, 간단한 셸 검색에서 시작한다.
  • 볼트 구조는 상위 허브를 비교적 안정적으로 유지하되, 세부 주제는 반복 사용으로 명확한 필요가 드러날 때만 확장한다.
